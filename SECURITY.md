# Security Policy

## Supported Versions

Currently, as Strand is in active early development, only the `main` branch is supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| main    | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Strand seriously. If you believe you have found a security vulnerability in Strand, please report it to us responsibly.

**Do not open up a public issue or create a public pull request for security vulnerabilities.**

Instead, please send an email to the project maintainers outlining the issue. Try to provide as much information as possible to help us reproduce the issue, such as:

- The type of issue (e.g. buffer overflow, SQL injection, cross-site scripting, etc.)
- Full paths of source file(s) related to the manifestation of the issue
- The location of the affected source code (tag/branch/commit or direct URL)
- Any special configuration required to reproduce the issue
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- The impact of the issue, including how an attacker might exploit the issue

We will try to respond to your report within 48 hours. If the issue is confirmed, we will release a patch as soon as possible and credit you for the discovery.
